Session Configuration

Customize how PerpetualCC sessions behave and interact with your codebase

AI Brain Configuration

Choose the AI model that powers decision-making. Rule-Based is fastest and requires no API keys. Gemini uses Google's API (requires GEMINI_API_KEY). Ollama runs locally on your machine.

Safety & Risk Settings

Controls how often you're asked to approve operations. Lower thresholds mean more autonomy but less control. Higher thresholds provide more safety checkpoints but require more interaction.
When enabled, operations deemed low-risk (like reading files, running tests) will be automatically approved without prompting. Disable for maximum control over all actions.

Execution Limits

The maximum number of action loops before a session automatically pauses. This prevents runaway sessions. Recommended range: 10-100 for most tasks. Set higher for complex projects. Please enter a value between 1 and 1000.
Maximum time a session can run before automatically stopping. Helps prevent forgotten sessions from running indefinitely. Set to 0 for no timeout (not recommended).

Advanced Settings

Controls the verbosity of session logs. Debug mode is helpful for troubleshooting. Warning or Error modes reduce noise but may hide useful information.
When enabled, sessions will simulate actions without actually executing them. Useful for testing configuration or understanding what the AI would do without making changes.
